# Echo Mirage · MU/TH/UR Doctrine

## Project Identity

Echo Mirage is evolving into a continuity-first AI operations platform focused on:

- orchestration
- embodiment
- persistent operational continuity
- trustworthy human-supervised coordination

The system prioritizes:

- continuity over spectacle
- orchestration over chaos
- visibility over hidden automation
- operational trust over fake theater
- local continuity over provider dependence

---

# MU/TH/UR

| Segment | Meaning | Operational Meaning |
|---|---|---|
| MU | MUTHUR Uses | MUTHUR currently has bounded operational control |
| TH | Thread Handoff | Control arbitration, scheduling, and negotiated transfer |
| UR | User Retake | Human override, interruption, and reclaim authority |

MU/TH/UR is not merely a name.

It is a control lifecycle doctrine.

---

# Core Operational Principle

All embodiment is temporary.

All control is negotiated.

Human override is absolute.

---

# Operational State Machine

```text
[UR]
User has control

UR → TH
handoff requested

[TH]
scheduler arbitration
permissions checked
capabilities scoped

TH → MU
control granted

[MU]
bounded execution active

MU → TH
task complete / timeout / interruption

TH → UR
control returned
```

# Current System Doctrine

The project is strongest when it remains:

- grounded
- visible
- coherent
- continuity-preserving
- operationally trustworthy

The project should avoid:

- decorative sci-fi theater
- fake AGI mythology
- hidden swarm chaos
- unrestricted automation
- fake logs or fake capabilities

---

# Human Sovereignty Rules

The human operator must always retain:

- interruption authority
- override authority
- visibility into execution
- ownership of control transfer

MUTHUR must never silently seize operational control.

All handoffs must remain visible and logged.

---

# Computer Use Doctrine

Computer-use capabilities must remain:

- bounded
- typed
- interruptible
- observable
- verifiable

Supported capabilities should:

- fail honestly
- report unsupported states gracefully
- avoid fake success states
- preserve operational trust
