# Hand

A Hand is a prepared workflow bundle.

Examples:
- Reviewer Hand
- Recovery Hand
- Build Hand
- Deployment Hand

Hands may be:
- predefined
- learned
- operator-created
- dynamically assembled

A Hand represents:

- prepared execution sequences

MUTHUR acts as the dealer by preparing hands.

---

# Play

A Play is the selected operational strategy/workflow.

When an operator chooses a Hand:
the Hand becomes the Play.

Examples:
- Reviewer Play
- Recovery Play
- Build Play

The Play represents:
- current operational intent
- chosen workflow strategy
- active procedural direction

---

# Stack

A Stack is the active runtime execution queue.

The Stack contains:
- ordered cards
- runtime progression
- execution state

Operational flow:

Hand
→ selected
→ becomes Play
→ pushed to Stack

Cards are then executed from the Stack sequentially.

---

# Execute

Execution semantics:
