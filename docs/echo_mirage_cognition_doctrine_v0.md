# Echo Mirage Cognition Doctrine v0

## Core Cognitive Hierarchy

```text
Context = current lived / active experience
Memory = associative long-term recall
Documents = official policy / doctrine / contracts
```

---

## Cognitive Stack

| Layer | Function | Authority |
|---|---|---|
| Context Window | current operational thought / active workflow | temporary |
| SQLite + Embeddings + Vector Retrieval | long-term associative memory / recall | advisory |
| Markdown Documents (.md) | doctrine / contracts / policies / identity anchors | authoritative |

---

## Active Operational Cognition

During operation:

```text
current context
+
nearest cosine / top-k relevant memory retrieval
+
doctrine documents
=
active operational cognition
```

MUTHUR operates primarily within:
- current experience
- active workflow
- live bridge conditions
- retrieved relevant memories

This allows:
- continuity
- posture restoration
- workflow recall
- doctrine grounding
without replaying infinite history.

---

## Dream Mode

Dream mode only runs when MUTHUR is not operating.

Trigger conditions include:
- no active lease
- no active workflow
- no pending user input
- no active stream
- no narration/TTS
- bridge quiet

Doctrine line:

```text
MUTHUR dreams only when the bridge is quiet.
```

---

## Dream Mode Responsibilities

Dream mode performs:
- indexing
- collating
- summarization
- ranking
- relationship reinforcement
- workflow extraction
- doctrine reference linking
- memory consolidation
- topology strengthening

Dream mode organizes memory/cache only.

Dream mode DOES NOT:
- rewrite doctrine
- modify contracts
- alter permissions
- silently change policy
- autonomously redefine operational law

## Architectural Principle

Documents are the source of truth.

Memory is:
- cache
- associative recall
- retrieval support
- continuity support

Documents remain:
- explicit
- inspectable
- versioned
- authoritative

This preserves:
- operational trust
- doctrine integrity
- bounded embodiment
- human sovereignty
